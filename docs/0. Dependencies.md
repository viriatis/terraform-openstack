## Check if you have an empty disk
One of the requirements of Openstack is to have at leat one dedicated empty disk to use as a Ceph OSD (Object Storage Daemon).

The requirement is:
* The disk must be completely empty (no partitions, no filesystem)
* It cannot be the OS disk
* Preferably identified by /dev/disk/by-id/ for stability

### If we want to wipe another disk, we can do it:

```shell
# Wipe everything from the disk
# Removes all filesystem signatures and magic bytes from the disk — things like "this is ext4", "this is LVM", "this is GPT". The -a means remove all signatures found.
sudo wipefs -a /dev/disk/by-id/your-disk-id

# Zero out the first and last few MB (removes partition tables and signatures)
# Writes 100MB of zeros to the beginning of the disk. This destroys:
## MBR partition table (first 512 bytes)
## GPT header (first few KB)
## Any bootloader
## Any filesystem superblock
sudo dd if=/dev/zero of=/dev/disk/by-id/your-disk-id bs=1M count=100

# The second dd with seek=...
## Writes 100MB of zeros to the end of the disk. This destroys the GPT backup header — GPT stores a backup of the partition table at the very end of the disk. If you only wipe the beginning, GPT can restore itself from the backup.
sudo dd if=/dev/zero of=/dev/disk/by-id/your-disk-id bs=1M count=100 seek=$(( $(sudo blockdev --getsz /dev/disk/by-id/your-disk-id) / 2048 - 100 ))

# Verify it's clean - should return nothing
## Just reads and displays any remaining signatures. If it returns nothing, the disk is pristine.
sudo wipefs /dev/disk/by-id/your-disk-id
## Lists partition information. If it returns nothing — no partitions exist.
sudo fdisk -l /dev/disk/by-id/your-disk-id | grep -i "type\|partition"
```

```shell
sudo apt update && sudo apt install -y snapd
sudo systemctl enable --now snapd

```

## Define FQDN and short hostname

### Check hostname
```shell
ubuntu@ubuntu-server:~$  hostname
ubuntu-server

ubuntu@ubuntu-server:~$  hostname -f
ubuntu-server
```

### Fix FQDN
Sunbeam requires a proper FQDN (Fully Qualified Domain Name) because Juju uses it to identify and register the machine in the cluster.

When Juju registers your physical server as a manual cloud machine, it uses the FQDN as the machine's unique identifier. If hostname -f returns just ubuntu-server instead of ubuntu-server.local or ubuntu-server.something.com, Juju can't properly resolve the machine and things break.

More on openstack architecture on: [Canonical Openstack Architecture](https://canonical.com/openstack/architecture)

```shell
sudo vim /etc/hosts
```

/etc/hosts should look like this:
```bash
127.0.0.1       localhost
127.0.1.1       ubuntu-server.local ubuntu-server
```
